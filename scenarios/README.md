# Scenario Examples

This directory contains cross-language examples that demonstrate a specific Agent Assembly capability end-to-end. Unlike the language directories, scenario examples focus on the **governance behavior** rather than a particular framework.

## What lives here

| Sub-project                        | What it demonstrates                                                        |
|------------------------------------|-----------------------------------------------------------------------------|
| `policy-enforcement/`              | Define and apply allow/deny policy rules; observe blocked and allowed calls |
| `approval-gates/`                  | Trigger human-in-the-loop approval for a high-risk tool call                |
| `audit-trace/`                     | Capture and inspect the audit log generated by a governed agent             |
| `budget-limits/`                   | Configure per-team call count and cost budgets; observe budget enforcement  |
| `sidecar-runtime/`                 | Intercept tool calls via the sidecar proxy without SDK code changes         |
| `live-core-enforcement/`           | Run the **real** SDK governed end-to-end by a live `aa-runtime` + `aa-gateway` (Python-only, Docker) |

## When to use scenario examples

Start with a language example (in `python/`, `node/`, or `go/`) if you want to see basic SDK integration in your ecosystem first.

Use a scenario example when you want to understand a specific governance capability — policy enforcement, approval gates, audit/trace, budgets, or sidecar interception — regardless of language.

See [docs/choosing-an-example.md](../docs/choosing-an-example.md) for the full decision guide.

## Expected sub-project structure

```text
scenarios/<scenario-name>/
  README.md         ← what it demonstrates, prerequisites, setup, run, expected output
  .env.example      ← template for any required config (never .env)
  docker-compose.yml ← optional: local gateway + agent stack for the scenario
  <language>/       ← one sub-directory per language variant (if cross-language)
```
